# PaperProof Academic Paper

This directory is for the academic-style PaperProof paper.

## Working Title

PaperProof: Sui-Native Artifact Infrastructure for Durable, Verifiable, and
Agent-Readable Knowledge Objects

## Purpose

This paper should present PaperProof as a system and protocol design rather
than as a product brochure. It should explain the problem, model, architecture,
security boundaries, evaluation evidence, and relationship to Sui and Walrus.

## Target Audience

- Hackathon judges who want technical depth.
- Researchers and protocol designers.
- Sui and Walrus ecosystem reviewers.
- Developers evaluating whether PaperProof is a serious protocol foundation.

## Suggested Files

- `paperproof-academic-paper.tex`
- `refs.bib`
- `figures/`
- `build/` or generated PDFs, if intentionally committed later.

## Suggested Structure

1. Abstract
2. Introduction
3. Motivation and Problem Statement
4. Design Goals
5. Artifact Model
6. Protocol Architecture
7. Sui Object Layer
8. Walrus Content Layer
9. SDK, Query, and Indexer Layer
10. Agentic Interface
11. Implementation and Mainnet Evidence
12. Security and Trust Boundaries
13. Limitations
14. Related Work
15. Conclusion

## Notes

Keep the tone formal, precise, and evidence-oriented. This paper can reuse
material from the yellow paper, UI design document, SDK documentation, slides,
and deployment records, but it should read as a coherent standalone paper.
